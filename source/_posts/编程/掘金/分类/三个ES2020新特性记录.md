h2&#123;font-size:20px&#125;.markdown-body h3&#123;font-size:18px&#125;&#125;</style><h3 data-id="heading-0">逻辑 OR 赋值 (||=)</h3>
<ul>
<li>逻辑OR赋值（x||=y），只在x为假时赋值</li>
</ul>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-keyword">const</span> a = &#123; <span class="hljs-attr">duration</span>: <span class="hljs-number">50</span>, <span class="hljs-attr">title</span>: <span class="hljs-string">''</span> &#125;;

a.duration ||= <span class="hljs-number">10</span>;
<span class="hljs-built_in">console</span>.log(a.duration);
<span class="hljs-comment">// expected output: 50</span>

a.title ||= <span class="hljs-string">'title is empty.'</span>;
<span class="hljs-built_in">console</span>.log(a.title);
<span class="hljs-comment">// expected output: "title is empty"</span>
<span class="copy-code-btn">复制代码</span></code></pre>
<p>返回x，在x为真时；返回y，在x为假时。</p>
<ul>
<li>不等于x=x||y</li>
</ul>
<h3 data-id="heading-1">可选链操作符(?.)Optional chaining</h3>
<ul>
<li>允许读取位于连接对象链深处的属性的值，而不必明确验证链中的每个引用是否有效。?. 操作符的功能类似于 . 链式操作符，不同之处在于，在引用为空(nullish ) (null 或者 undefined) 的情况下不会引起错误，该表达式短路返回值是 undefined。与函数调用一起使用时，如果给定的函数不存在，则返回 undefined。</li>
</ul>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-keyword">const</span> adventurer = &#123;
  <span class="hljs-attr">name</span>: <span class="hljs-string">'Alice'</span>,
  <span class="hljs-attr">cat</span>: &#123;
    <span class="hljs-attr">name</span>: <span class="hljs-string">'Dinah'</span>
  &#125;
&#125;;

<span class="hljs-keyword">const</span> dogName = adventurer.dog?.name;
<span class="hljs-built_in">console</span>.log(dogName);
<span class="hljs-comment">// expected output: undefined</span>

<span class="hljs-built_in">console</span>.log(adventurer.someNonExistentMethod?.());
<span class="hljs-comment">// expected output: undefined</span>
<span class="hljs-built_in">console</span>.log(adventurer.cat?.name)
<span class="copy-code-btn">复制代码</span></code></pre>
<ul>
<li>引用对象，寻找潜在子属性时，不必考虑父级元素是否为null或者undefined，而不必考虑抛出错误，无需检验父级的状态</li>
</ul>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-keyword">let</span> nestedProp = obj.first?.second;
<span class="copy-code-btn">复制代码</span></code></pre>
<p>等价于</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-keyword">let</span> temp = obj.first;
<span class="hljs-keyword">let</span> nestedProp = ((temp === <span class="hljs-literal">null</span> || temp === <span class="hljs-literal">undefined</span>) ? <span class="hljs-literal">undefined</span> : temp.second);
<span class="copy-code-btn">复制代码</span></code></pre>
<ul>
<li>引用方法，调用一个可能不存在的方法时也可以使用可选链。这将是比较有用的，例如，当使用一个 API 的方法可能不可用时，因为当前用户的设备不可用支持该功能。</li>
<li>函数调用时如果被调用的，使用任选链可以使表达式自动返回undefined而不是方法抛出一个异常。</li>
</ul>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-keyword">let</span> result = someInterface.customMethod?.();
<span class="copy-code-btn">复制代码</span></code></pre>
<ul>
<li>
<p>注意：如果存在一个名且不是函数，使用 ?.属性会产生一个 TypeError异常（x.y is not a function）。</p>
</li>
<li>
<p>注意：如果someInterface 自己是null或者undefined，异常 TypeError还会被抛出 someInterface is null 如果你希望 someInterface也为null或者undefined，那么你需要像这样写 someInterface?.customMethod?.()</p>
</li>
<li>
<p>事件处理器</p>
</li>
</ul>

<h3 data-id="heading-2">空值合并运算符??</h3>
<ul>
<li>空值合并操作符（??）是一个逻辑操作符，当左侧的操作数为 null 或者 undefined 时，返回其右侧操作数，否则返回左侧操作数。</li>
<li>与逻辑或操作符（||）不同，逻辑或操作符会在左侧操作数为假值时返回右侧操作数。也就是说，如果使用 || 来为某些变量设置默认值，可能会遇到意料之外的行为。比如为假值（例如，'' 或 0）时。见下面的例子。</li>
</ul>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-keyword">const</span> foo = <span class="hljs-literal">null</span> ?? <span class="hljs-string">'default string'</span>;
<span class="hljs-built_in">console</span>.log(foo);
<span class="hljs-comment">// expected output: "default string"</span>

<span class="hljs-keyword">const</span> baz = <span class="hljs-number">0</span> ?? <span class="hljs-number">42</span>;
<span class="hljs-built_in">console</span>.log(baz);
<span class="hljs-comment">// expected output: 0</span>
<span class="copy-code-btn">复制代码</span></code></pre>
<ul>
<li>用空值合并运算在逻辑正确的前提下，代码更加简洁。空值合并运算符 与 可选链 相结合，可以很轻松处理多级查询并赋予默认值问题。</li>
</ul></div>
