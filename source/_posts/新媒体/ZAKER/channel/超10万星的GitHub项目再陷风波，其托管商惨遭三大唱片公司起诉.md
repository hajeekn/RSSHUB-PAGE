<p>整理 | 于轩 责编 | 张红月</p><p>最近，声称为开发者 " 挺身而出 "，在 GitHub 上拥有超十万星的 youtube-dl 正深陷起诉风波。目前，围绕这个开源项目长达数年的争议已经转移到了一个新的层面，其网站托管商 Uberspace 在德国遭到了索尼、环球和华纳三大唱片巨头的起诉。</p><p></p><div class="img_box" data-height="321" data-width="676" src="https://cors.zfour.workers.dev/?http://zkres1.myzaker.com/202201/61e7b7a28e9f09269914cc11_1024.jpg" referrerpolicy="no-referrer"></div></div>Uberspace 官网截图<p></p><p>youtube-dl 实际上是一个为其他程序提供各种功能的软件库，除了下载使用外，也可以通过命令行直接使用该软件库里的内容。同时，作为一个广受欢迎的开源下载管理器，它允许用户从 YouTube、优酷、B 站等上百个视频门户网站下载视频和音频。</p><p><strong>项目因起诉而被移除</strong></p><p>早在 2020 年 10 月，托管 youtube-dl 代码的 GitHub 收到了美国唱片业协会（RIAA）的 DMCA（《数字千年版权法案》） 通知，一家唱片公司对 youtube-dl 提起了诉讼。这家公司在诉讼中表示，youtube-dl 给它们的唱片业务带来了风险，称其规避了 YouTube 的技术保护措施。因为 Youtube 只通过有效的 DRM 加密 ( Digital Rights Management ) 来保护付费内容，大部分 Youtube 目录只提供了所谓的 " 滚动密码 "。</p><p>虽然 youtube-dl 并没有尝试规避 DRM 保护，但 RIAA 引用了 DMCA 的反规避条款，称 youtube-dl 工具可用于从 YouTube 下载其艺术家未授权的音乐作品，而这违反了版权。不过当时在美国没有相似可供参考的案例，所以 RIAA 引用了德国汉堡市法院在一个类似案件中的裁决，该德国法院在 2017 年认定这是非法的。该裁决认为，根据欧盟法律，YouTube 的 " 滚动密码 " 被视为一项有效的技术保护措施，任何试图规避它的行为都将构成侵权。所以 GitHub 不得不短暂移除了所有 youtube-dl 项目。但随后不久，GitHub 迫于开发者的疯狂抵制，最终又恢复了该软件并投入了 100 万美元建立防御基金。</p><p>实际上，Uberspace 托管的网站 https://youtube-dl.org 只相当于程序库的一张名片，代表的是 GitHub 上的项目页面，它并不托管工具本身。</p><p><strong>无视警告，又被起诉</strong></p><p>其实在 GitHub 案裁决结果出来的几周前，德国就已经开始行动了。所以在该项目被移除后不久，律师事务所 Rasch Rechtsanwalte 与几个主要的音乐行业参与者合作，向当地的托管服务商 Uberspace 发出警告。警告称，该提供商有共谋、助手和麻烦制造者责任，应该避免托管该网站。并表示其是代表索尼娱乐、华纳音乐集团和环球音乐的德国分公司发送的。</p><p>律师事务所老板克莱门斯 · 拉施（Clemens Rasch）已经处理过多次音乐行业的纠纷，他早在 2020 年 9 月发出警告之前，就让 Uberspace 的老板乔纳斯 · 帕斯奇（Jonas Pasche）提交一份停止声明，否则将被起诉。但这位企业家立即通过他的律师拒绝了这一点，认为这完全没有道理。帕斯奇当时表示：" 软件本身并没有托管在我们的系统上，所以说实话，无论如何让我们参与这个问题都是相当荒谬的——一个专门研究信息技术法律的律师应该更清楚。" 他强调，下载 YouTube 的内容，只要不需要规避版权保护，并且视频不是明显非法制作的，这在联邦司法部看来，一般来说都是允许的。</p><p>警告无果后，律师事务所 Rasch Rechtsanwalte 代表客户 Mia、Wincent Weiss 和 Robin Schulz 在德国起诉了纳斯 · 帕斯奇，罪名是 " 共谋、共犯和麻烦制造者责任 "。根据 Netzpolitik.org 的一份报告，这三位艺术家的唱片公司在让用户访问 youtube-dl 之前，向 Uberspace 的老板 Jonas Pasche 投了钱。因此，被告在很大程度上促成了用户可以获取那些并未提供下载内容的事实。下载时也会绕过 " 有效的技术保护措施 "。另外，Uberspace 还接受了开源项目的付款。</p><p><strong>行业还未有基本判断标准</strong></p><p>对此，纳斯 · 帕斯奇在给三家公司的回复中表示：YouTube 没有实施任何旨在防止下载版权内容的技术措施，客户也不依赖于任何通过数字版权控制管理（DRM）的技术保护。从一个合理的 YouTube 用户角度来看，这听起来像是制作私人副本的一个友好邀请。这一选择将由收集协会 Gema 支付相应费用来补偿，youtube-dl.org 上的下载链接也只是 " 重定向到 GitHub 上的实际下载源 "。</p><p>尽管有这些反驳，根据报告，投诉很大程度上类似于警告。Pasche 的律师怀疑 " 诉讼的目的是为了实现一种基本的判断 "。如果该行业侥幸逃脱，如果被投诉内容的非法性在法律上有争议，主机提供商也可能被迫采取行动。原则上，像 Uberspace 这样的在线服务受到所谓的提供者特权的保护，对用户可能的违法行为不负直接责任。如果被告知服务上有非法内容，但却无动于衷，这对其来说就有法律意义。</p><p><strong>Uberspace 并未就此屈服</strong></p><p>youtube-dl 是否为侵犯版权提供便利，多年来一直是人们热议的话题。RIAA 根据《数字千年版权法案》 ( DMCA ) ，最初于 2020 年秋季在 GitHub 上封锁了 Python 库。在强烈抗议后，微软运营的平台稍晚恢复了目录。其辩解说，根据自己律师的评估，该软件没有违反 DMCA。</p><p>自由权利协会（GFF）的朱莉娅 · 雷达（Julia Reda）说 :" 即使 YouTube 不赞许视频下载的功能出炉，这些视频也没有提供版权保护。在用户界面上隐藏下载网址并不等同于技术性的复制保护，规避这种保护是版权法所禁止的。这位版权专家和前欧盟议员说，汉堡地区法院当时的判决没有认识到 " 自己下载一段 YouTube 视频只需要基本的技术知识 "。</p><p>Reda 警告称，如果托管提供商被警告覆盖，他们可能会成为进一步的附带损害 :" 简单地遵守警告或删除通知，而不坚持认为相关内容的非法性实际上是由最高法院的意愿决定的，这很容易做到。这一方面，Uberspace 没有简单地屈服该要求，这是受欢迎的做法。"</p><p>Uberspace 目前正在对抗此事，并在积极准备辩护。如果 Uberspace 在这一过程中败诉，可能会给公司带来巨大损失。物质价值为 10 万欧元，加上法律费用，包括对方的费用。仅此一项就可能占到争议价值的百分之十左右。youtube-dl 开发人员在其网站上写道：项目主持人已经花了数千欧元进行法律辩护。</p><p><strong>可选方案不计其数</strong></p><p>不过，用户不必担心下载 YouTube 视频受影响，至少短期内不用。youtube-dl 的源代码和可执行文件不在 Uberspace，而是在属于微软的 GitHub 里。该厂商在审查了该行业的 DMCA 并发现其不合理后，恢复了项目的代码库。此外还有一些分叉，即拆分出来的 youtube-dl 的开源变种，以及无数其他工具，可以将 YouTube 内容 " 铲 " 到自己的硬盘上。</p><p>令 Uberspace 老板纳斯 · 帕斯奇觉得奇怪的是，谷歌搜索 "YouTube 下载 " 时，首先显示了一个与搜索结果分开的信息框，上面有一个来自 youtube-dl 的维基百科条目标记。纳斯 · 帕斯奇表示：" 如果 Youtube 如此热衷于防止下载，谷歌作为所有者至少可以确保它不在显著位置宣传（youtube-dl）可能是最普遍的下载工具。"</p><div id="recommend_bottom"></div><div id="article_bottom"></div>
