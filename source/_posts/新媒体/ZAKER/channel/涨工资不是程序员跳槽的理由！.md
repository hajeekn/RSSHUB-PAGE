<p>整理 | 丁广辉 </p><p>在最近二十年中，全球行业都经历了一次数字化变革，各行各业对于技术开发的比重越来越高，而作为技术开发核心的开发人员们对于一个企业的未来发展也变得越来越重要。因此各企业对于技术人才的竞争变得火热，并且这个热度一年高过一年。</p><p>据技术问答网站 Stack Overflow 发布的 THE OVERFLOW 调查显示，他们调查了几乎所有主要的招聘网站，加起来一共有超过 7 万个技术职位空缺。然而仅有 25% 的开发人员在积极的找工作，54% 左右的开发人员原意接受新的工作机会，但并不会主动去找，却有近 20% 的开发人员表示对新的工作机会不感兴趣。也就说明有 74% 的开发人员不会去找新工作。这就让世界上几乎所有的技术人才招聘团队面临了一个难题：怎样吸引技术人才？让我们一起看下影响开发人员寻找工作的因素有哪些。</p><p></p><div class="img_box" id="id_imagebox_0" onclick><div class="content_img_div perview_img_div"><img class="lazy 希望能学习使用新的技术，36% 希望能更好地平衡工作与生活，35% 则是在寻求能力增长或者成为领导者的机会。</p><p></p><div class="img_box" id="id_imagebox_1" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_1" 77%）都会将新企业的这些方面与现在的企业进行比较，他们在考虑留下与考虑离开时所思考的因素几乎是一样的，在为什么要离开当前企业的原因中除了薪水（69%）；工作灵活性（61%）和学习机会（53%）是开发人员考虑的最重要因素。<p></p><p>同样，当开发人员考虑留在目前的工作岗位时，他们所考虑的依旧时上述的三个因素，但不同的是，这次对于工作的灵活性的要求 ( 65% ) 甚至胜过薪水 ( 59% ) 和学习机会 ( 56% ) 。事实上，在涉及工作时间和远程工作时，在被调查者中优先考虑工作灵活性这件事上无论每个年龄段都完全一致，但 00 后除了工作灵活性还很看重在新企业的学习机会。</p><p></p><div class="img_box" id="id_imagebox_2" onclick><div class="content_img_div referrerpolicy="no-referrer"></div></div><strong>开发人员重视工作中的灵活性和学习机会</strong><p></p><p>那么，是因为什么原因让企业能吸引到技术人才呢。超过 53% 认为是企业在招聘时优先考虑开发人员的经验，其次是企业薪资透明（41%），第三是企业员工有向自己团队以外的人学习的机会（40%）。当开发人员准备加入一家企业时，他们还再寻找什么能吸引自己。35% 的人认为完善的员工培训很吸引人，有 33% 的人认为企业里有专家、高手更能吸引自己。综上所述，开发人员更重视工作的灵活性和更多的学习机会。做到这些企业才会更能吸引开发人才。</p><p></p><div Overflow，那么会有 60% 的人不选择该企业。企业没有足够的资源来让员工感到对工作的自信，则有 54% 的人认为企业没有对自己没有吸引力。在灵活性方面，56% 的开发人员表示如果企业的工作时间不是弹性的，那么该企业也会被开发人员拒之门外，除此之外如果企业要求他们在办公室工作，他们会觉得企业没吸引力。<p></p><p></p><div class="img_box" ChatOps 并不能成为他们拒绝一家企业的原因，只有 8-12% 的人表示，没有这些工具会让一家企业失去对他们的吸引力。而企业就算缺乏那些遗留的维基系统，也仅有 20% 的人表示这会让自己觉得企业没吸引力。<p></p><p><strong>信誉就是一切</strong></p><p>被动地看开发人员是否想为一家企业工作，往往开发人员会再相同的四个方面找到他们初步想要的工作：企业的个人网络、媒体、企业的一些博客和文化视频内容以及评论。当开发人员偶然遇到想要去工作的企业时，他们会先询问朋友或家人，然后阅读有关该企业的媒体报道。26% 的开发人员表示，他们从网站上的广告中发现了可以考虑入职的企业。对于积极寻找新工作的人，他们倾向于先看企业评论，再看媒体报道，然后才会从其他方面考虑是否入职。</p><p></p><div class="img_box" id="id_imagebox_5" onclick><div class="content_img_div perview_img_div"><img class="lazy 的开发人员退出面试过程的主要原因是他们不喜欢技术栈。其他主要原因还包括面试流程混乱 ( 24% ) 、面试问题奇怪 ( 24% ) 、雇主评估差 ( 24% ) 以及无法找到足够的信息了解在这家企业工作的情况 ( 22% ) 。</p><p>企业要是想确保自身的技术栈不会让来应聘的开发人员望而却步，就需要满足技术人员的需求。在 Stack Overflow 对超过 83,000 名开发人员的年度调查中，他们深入研究了开发人员用的最流行的语言和技术，以及开发人员最想使用的是什么。Stack Overflow 发现流行的语言和框架都很灵活。例如，Rust 连续第六年成为最受欢迎的语言，而 Python 是第五年最受欢迎的语言，这两种语言都因灵活而受欢迎。新产品 Svelte 自进入榜单的第一年就成为最受欢迎的框架，React 则是四分之一的开发人员渴望想要的框架。</p><p><strong>总结</strong></p><p>无论是工作场所政策还是技术栈，开发人员都渴望工作的灵活性和一个学习新东西的机会。如果他们在目前的岗位上找不到这样的工作，他们当然会去别处寻找。在目前有超过 7 万个技术职位的空缺，有 20% 的开发人员正在积极寻找新工作，人才争夺战不会很快平息。希望其他技术岗位也会效仿。</p><div
