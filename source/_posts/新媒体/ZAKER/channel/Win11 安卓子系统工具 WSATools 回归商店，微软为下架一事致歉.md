<p>IT 之家 11 月 21 日消息，半个月前，微软悄悄从 Windows 应用商店中删除了 WSATools 工具 —— 这是为 Windows 11 安卓子系统（WSA）提供 APK 应用本地安装的程序，但微软本身没有提供任何明确的原因。</p><p></p><div class="img_box" id="id_imagebox_0" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_0" data-original="http://zkres1.myzaker.com/202111/618f9c288e9f0928c26566d3_1024.jpg" data-height="555" data-width="808" src="https://cors.zfour.workers.dev/?http://zkres1.myzaker.com/202111/618f9c288e9f0928c26566d3_1024.jpg" referrerpolicy="no-referrer"></div></div>经过一些争议后，微软似乎已经努力与 WSATools 开发者 @Franco 解决了这个问题。外媒发现目前该应用已经重新上架了 Windows 官方的微软商店。此外，据开发者所说，微软之前还特意写了一封邮件向他道歉，并说明了下架的原因。<p></p><p></p><div class="img_box" id="id_imagebox_1" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_1" data-original="http://zkres1.myzaker.com/202111/619a22d68e9f09627156fb12_1024.jpg" data-height="554" data-width="600" src="https://cors.zfour.workers.dev/?http://zkres1.myzaker.com/202111/619a22d68e9f09627156fb12_1024.jpg" referrerpolicy="no-referrer"></div></div>IT 之家了解到，Windows 11 安卓子系统 ( Windows Subsystem for Android ) 包括 Linux 内核和基于 Android 开源项目（AOSP）版本 11 的 Android 操作系统。该子系统在 Hyper-V 虚拟机中运行，就像 Linux 子系统一样，可以将 AOSP 环境中 App 的运行时和 API 映射到 Windows 图形层、内存缓冲区、输入模式、物理和虚拟设备以及传感器，可以在英特尔、AMD、高通的 CPU 上运行。<p></p><p></p><div class="img_box" id="id_imagebox_2" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_2" data-original="http://zkres2.myzaker.com/202111/619a22d68e9f09627156fb13_1024.jpg" data-height="494" data-width="1440" src="https://cors.zfour.workers.dev/?http://zkres2.myzaker.com/202111/619a22d68e9f09627156fb13_1024.jpg" referrerpolicy="no-referrer"></div></div><p></p><div id="recommend_bottom"></div><div id="article_bottom"></div>
