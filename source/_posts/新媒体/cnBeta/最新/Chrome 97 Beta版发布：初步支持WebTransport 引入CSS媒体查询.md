在 Chrome 96 稳定版发布之后，开发团队的重心已经偏移到 Chrome 97 版本中。在今天发布的 <a href="https://www.google.com/intl/zh-CN/chrome/beta/" target="_blank">Chrome 97 Beta</a> 版本中，最引人注意的是添加了对 WebTransport 的初步支持。WebTransport 是一个类似于WebRTC数据通道的协议框架，但主要是用于受网络安全模型限制的客户端，使用安全、多复式传输与远程服务器进行通信。<br>
 <p style="text-align: left;">WebTransport使用HTTP/3协议进行双向传输。与基于TCP的WebSockets不同，WebTransport依赖于类似UDP的数据包和可取消的流。通过W3.org上的W3C工作草案了解更多关于WebTransport的信息。</p><p style="text-align:center"><a href="https://static.cnbetacdn.com/thumb/article/2021/0831/10fe8576d94a11e.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2021/0831/10fe8576d94a11e.jpg" referrerpolicy="no-referrer"></a></p><p style="text-align: left;">Chrome 97 Beta 版还引入了 CSS 媒体查询，以了解设备的显示屏是否支持高动态范围（HDR）。此外，在开发者方面，Chrome 97 Beta 版还引入了 findLast 和 findLastIndex 的 JavaScript 方法，作为 Array.prototype 的一部分。正如方法名称所暗示的那样，它们是为了轻松找到数组中最后一个匹配的元素，而不是为了轻松找到数组中最后一个匹配的元素而必须反转数组或进行其他非繁琐的修改。</p><p style="text-align: left;"><strong>测试版 64 位
