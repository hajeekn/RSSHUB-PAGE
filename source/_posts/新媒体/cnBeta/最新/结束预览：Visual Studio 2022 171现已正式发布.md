<strong>在几周前放出了 Preview 2 预览之后，微软终于在今日正式发布了 Visual Studio 2022 17.1 。</strong>作为一套功能强大的集成开发环境（IDE），之前一个多月的预览功能已基本融入正式版，此外微软整合了从最终用户那里收集到的大量反馈。<br>
 <p><img src="https://static.cnbetacdn.com/article/2022/0216/24cb5de97a0dde4.jpg" alt="1.jpg" referrerpolicy="no-referrer"></p><p style="text-align: center;">（图 via <a href="https://www.neowin.net/news/visual-studio-2022-171-is-now-generally-available/" target="_self">Neowin</a>）</p><p>首先，随着 VS 2022 17.1 的正式到来，其现已默认启用文件中的索引查找，以带来更畅快的搜索体验。</p><p>若出于某种原因，你想要禁用这项功能，可移步至“工具（Tools）→ 选项（Options）→ 环境（Environment）→ 预览功能（Preview Features）”反选。</p><p>其次，一项新功能可在每次 IDE 窗口失焦时，尝试保存 IDE 中的每个“脏”文档（dirty document），比如项目 / 解决方案。</p><p>此外有一些 Git 功能增强，例如比较分支（compare branches）、签出提交（checkout a commit）、查看查询请求（review pull requests）等。</p><p>开发者还可在使用 RTOS 对象时，获得可用寄存器及其映射内存位置的一些可视化信息，以及利用解决方案过滤器（Solution Filters）。</p><p><strong>微软写道：</strong></p><blockquote><p>该功能允许开发者选择要加载的项目，以避免每次都加载大量的内容。你可选择加载单个项目、或带有整个依赖关系树的项目。</p><p>不过有用户留意到，当添加新项目、或依赖关系图发生变化时，依赖关系树 / Solution Filters 可能会过期。</p><p>为此，VS 2022 17.1 版本中添加了一种更简单的方法，来为 Solution Filters 中的所有项目加载整个依赖关系树。</p><p>这将引入任何添加的新依赖项，以确保开发者拥有所需的所有项目。</p><p>您可通过单击解决方案资源管理器（Solution Explorer）上下文菜单中的更新项目依赖（Update Project Dependecies），来检查任何新的依赖项。</p></blockquote><p>最后，微软建议专业和企业客户开始研究如何利用 VS 2022 17.1 中的新功能，尤其是那些处于 17.0 长期支持服务（LTSC）通道上的功能。</p><p>之后微软将把重心放到已经开启预览的 VS 2022 17.2 上，可知其包含了更多 Git 增强、针对 MAUI 开发的 Bug 修复、对新 C# 11 重构的支持、以及在本地处理数据时的相关改进。</p><p><img src="https://static.cnbetacdn.com/article/2022/0216/9e7fd1dc870a023.png" referrerpolicy="no-referrer"></p><p><strong>Visual Studio 2022 17.1 下载地址：</strong></p><blockquote><p><a href="https://visualstudio.microsoft.com/zh-hans/downloads/" _src="https://visualstudio.microsoft.com/zh-hans/downloads/" target="_blank">https://visualstudio.microsoft.com/zh-hans/downloads/</a><br></p></blockquote><div class="article-relation"><p><strong>相关文章:</strong></p><p><a href="https://www.cnbeta.com/articles/soft/1222953.htm" target="_blank">Visual Studio 2022 17.1 Preview 2发布：带来Git、C++和.NET增强功能</a></p></div>
