2016 年 2 月 16 日正式亮相的 Vulkan 1.0，即将迎来六周年的生日。<strong>而通过为期两年的努力，Khronos Group 已于本周二放出了 Vulkan 1.3 的重大更新。</strong>Phoronix 指出：除了将更多扩展挪到核心，它还引入了新的“配置文件”（profiles）概念。<br>
 <p><img src="https://static.cnbetacdn.com/article/2022/0126/dc68db71a40cf47.jpg" alt="1.jpg" referrerpolicy="no-referrer"></p><p>距离 Vulkan 1.2 的发布，已经过去两年时间。而随着 Vulkan 1.3 的到来，我们发现它迎来了生态系统方面的重大改变。</p><p>据悉，Vulkan 1.3 仍属于核心规范，但对于超越了该扩展的中高端移动 / 桌面设备来说，这也意味着一个全新的里程碑。</p><p><img src="https://static.cnbetacdn.com/article/2022/0126/da4c5ba12399d7f.jpg" alt="2.jpg" referrerpolicy="no-referrer"></p><p>因为新规划的路线图指南，有助于为如今的现代高端图形处理器提供集中式的支持，并且引入了可自定义的“配置文件”新概念。</p><p>配置文件为所需实现的特定 Vulkan 功能提供了概述，而首个“官方配置文件”，就是来自 Google 的 Android Baseline 2021 Profile 。</p><p><img src="https://static.cnbetacdn.com/article/2022/0126/c20efac42f28480.jpg" alt="3.jpg" referrerpolicy="no-referrer"></p><p>其主要针对 Vulkan 1.0 以上的特定功能，预计会存在于绝大多数的 Android 设备中。而且它不仅适用于现代设备，也能够在许多较旧的设备上提供支持。</p><p>其次是 OpenGL ES 3.1 类硬件，也将继续支持 Vulkan 1.3 的核心规范。一些激动人心的功能，现也作为核心扩展而引入，比如动态渲染（Dynamic Rendering）。</p><p><img src="https://static.cnbetacdn.com/article/2022/0126/0127582a018b1ca.jpg" alt="4.jpg" referrerpolicy="no-referrer"></p><p>另外还有围绕缓冲区设备地址、点积、KHR Synchronization2，以及其它总计两打新加入的扩展。</p><p>预计各大 GPU 厂商会从即日起提供测试版驱动程序，且 Vulkan 1.3 的 Git 将很快迎来开源的 Mesa 驱动。</p><p><img src="https://static.cnbetacdn.com/article/2022/0126/f873968ce1a4fcc.jpg" alt="5.jpg" referrerpolicy="no-referrer"></p><p>当谈到新的“路线图里程碑”（Vulkan Roadmap 2022 Milestone）时，Khronos Group 还指定了 Vulkan 1.3 核心之外的一组新功能。</p><p>所有这些新增功能和更高的要求，都是切实地想要更好地利用现代 GPU 在桌面 / 移动设备端的潜力。同时为改善开发体验，而为中高端 GPU 设立新的目标基线。</p><p><img src="https://static.cnbetacdn.com/article/2022/0126/47d38efed7e11ec.jpg" alt="6.jpg" referrerpolicy="no-referrer"></p><p>作为路线图更改的一部分，旧有的 GPU 将继续支持 Vulkan 1.3+ 版本，同时开发团队正在对未来的 GPU 芯片设计工作开始产生影响。</p><p>至于 Vulkan API 的更长期规划，可知配置文件规范（Profile Specifications）允许轻松概述如何实现预期的特定扩展和其它要求。</p><p><img src="https://static.cnbetacdn.com/article/2022/0126/e0d427eba35e094.jpg" alt="7.jpg" referrerpolicy="no-referrer"></p><p>以 Google 为例，该公司就试图在其平台上，让 Vulkan 驱动程序在细粒度上更进一步。</p><p>比如能够解析基于 JSON 的配置文件，且即将推出的 Vulkan SDK 更新将允许用户 / 开发者轻松检查他们的驱动程序或系统是否符合给定的配置文件的要求。</p><p><img src="https://static.cnbetacdn.com/article/2022/0126/4da4d5c81311397.jpg" alt="8.jpg" referrerpolicy="no-referrer"></p><p>至于最新发布的 Vulkan 1.3，可简单归纳其促进了核心方面的一些扩展。随着时间的推移，我们很是期待后续发布的其它配置文件的有趣证明。</p><p><img src="https://static.cnbetacdn.com/article/2022/0126/54b8ef6b8ed0d68.jpg" alt="9.jpg" referrerpolicy="no-referrer"></p><p>从 2022 路线图来看，Vulkan 提升了我们对于现代高端 GPU 的预期，且 Vulkan 配置文件规范能够实现量身定制的功能建议，比如 Android ISV / IHV 。</p><p><img src="https://static.cnbetacdn.com/article/2022/0126/be08a7d227817f7.jpg" alt="10.jpg" referrerpolicy="no-referrer"></p><p>感兴趣的朋友，可移步至 Khronos.org <a href="https://khronos.org/" target="_self">官网</a>，了解与 Vulkan 1.3 有关的更多细节。</p>
