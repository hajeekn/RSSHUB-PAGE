<strong>微软正准备为 Dev Channel 的 Insider 测试者们推送一批新功能，其中就包括“开始”中的应用程序文件夹、新手势、以及新颖的 Snap Bar 使用体验。</strong>鉴于上一个 Dev Channel 版本已经过去了两周，我们推测上述改进也将差不多时候到来。<br>
<p><img src="https://static.cnbetacdn.com/article/2022/0212/9cc92eb9a9b1dc0.jpg" referrerpolicy="no-referrer"></p><p>Windows Central 从知情人士处获悉，<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://www.microsoftstore.com.cn/" target="_blank">微软</a>上周向 Windows 11 的公共预览分支（rs_prerelease）引入了一系列功能和更改。</p><blockquote><p>预计在接下来几周内，Dev Channel 的 Insider 测试者们将迎来可在‘开始’菜单中创建应用程序文件夹的能力。</p><p>此外支持在任务栏上拖放文件、为旧式标题栏引入新的亚克力 / 模糊效果、名为‘Snap Bar’的新体验，以及通过触摸开打开‘开始’和‘快速设置’面板的新手势。</p></blockquote><p>首先，“开始”中的应用程序文件夹，是 Insider 测试者们强烈要求的一项功能。而即将到来的体验，也完全符合大家的预期。</p><blockquote><p>你可以将某个应用程序图标拖动到另一个图标上方，以创建一个可单击进入到文件夹，从而更好地组织‘开始’菜单布局。</p></blockquote><p>其次，“Snap Bar”据说引入了另一种可将多个应用程序窗口、捕获并放置于 Windows 11 内置网格系统的方法。</p><blockquote><p>当前我们有两种在屏幕上执行此操作的方法 ——（1）通过将鼠标悬停在窗口空间上时出现的下拉菜单；（2）将应用程序窗口拖动到屏幕的顶部或侧面。</p><p>至于 Snap Bar 带来的第三种 UX 方案，则是当永不在显示器周围抓取或移动窗口时，都会有一个从屏幕顶部往下拉的‘面板栏’出现。</p></blockquote><p style="text-align: center;"><img src="https://static.cnbetacdn.com/article/2022/0212/adbc6cf9e6c3b73.jpg" referrerpolicy="no-referrer"></p><p>其它 UI 改进涵盖了为标题栏引入洗的呢亚克力 / 模糊效果，它将取代当今可在大多数 Win32 桌面应用程序中看到的白色标题栏设计。</p><blockquote><p>熟悉 Windows 7 / Vista 时代 Windows Aero 体验的朋友，一定不会对此感到陌生。</p><p>与此同时，微软为其融入了全新的 Fluent 设计风格，类似 Windows 11 上其它模糊的半透明界面。</p></blockquote><p>平板体验方面，Windows 11 也将迎来手势控制方面的一项重大改进。比如从屏幕底部向上滑动，即可访问“开始”菜单和“快速设置”面板。</p><blockquote><p>从系统托盘向上滑动将打开‘快速设置’，而在其它任何地方向上滑动，则会唤出‘开始’菜单（就像在 Windows 10X 上一样）。</p></blockquote><p>最后，Windows 11 任务栏也将补足首发时短缺的某些功能，比如支持拖放操作，将文件附加到特定的应用程序中打开。</p><blockquote><p>此外对于溢出的应用程序图标，微软也将带来更加友好的用户界面，并且能够在平板模式下自动隐藏任务栏。</p></blockquote>
