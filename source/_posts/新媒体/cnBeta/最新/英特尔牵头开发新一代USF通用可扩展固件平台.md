<strong>英特尔刚刚宣布了与“通用可扩展固件”（USF）有关的最新消息，可知其旨在简化、扩展从边缘计算到云硬件的固件开发。</strong>据悉，新发布的 USF 规范草案基于 UEFI 和 ACPI 等现有行业标准之上，在 SoC、平台和操作系统之间引入了新的抽象和域边界，能够更好地适应新的芯片与平台技术。<br>
 <p><img src="https://static.cnbetacdn.com/article/2021/1118/23d7105ad3bff99.jpg" referrerpolicy="no-referrer"></p><p style="text-align: center;">（来自：Intel USF <a href="https://www.intel.cn/content/www/cn/zh/developer/articles/technical/universal-scalable-firmware.html" target="_self">官网</a>）</p><p>除了较现有固件更高的安全性，<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://intel.jd.com/" target="_blank">英特尔</a>还希望将通用可扩展固件（USF）的应用范围推广到系统固件之外，比如该公司的独显产品线。</p><p>已知关键计划功能 / 组件涵盖了可跨不同操作系统和引导加载程序工作的通用有效负载，具有简化的 ACPI 支持、Rust 编程语言接口、YAML 配置、以及 SoC FSP 等。</p><p>推广开来之后，英特尔希望 USF 能够进一步降低固件开发成本，同时提升固件质量和安全性、以及推动固件领域的其它创新。</p><p>需要指出的是，尽管通用可扩展固件主打“开放”特性，但英特尔也承认 USF 其实由外部行业规范 + 内部规范组成，涵盖了硬件和固件之间的 SoC 结构和内部接口。</p><p>在这点上，Phoronix 猜测新固件规范与英特尔开源 FSP、或使 USF 成为完全开源的固件堆栈，并不会有任何关系。</p><p>最后，英特尔表示该公司打算启用关键的开源项目，包括 TianoCore、Coreboot 和具有 USF 支持的 Slim 引导加载程序。</p><p>感兴趣的朋友，可移步至 GitHub 项目主页（传送门 <a href="https://github.com/universalscalablefirmware" target="_self">1</a>），或直接查看 USF 规范文档（传送门 <a href="https://universalscalablefirmware.github.io/documentation/" target="_self">2</a>）。</p>
